04 / 23 · 现状诊断
CRISIS
当前航天自主控制的两条技术路线——完全割裂

模型驱动 vs 数据驱动

01

模型驱动

完全建立在轨道动力学之上。最优控制、MPC（模型预测控制）、轨迹优化。可解释、可靠、工程验证充分。但缺乏学习能力——面对未知环境手足无措。

02

数据驱动

神经网络、强化学习直接学轨迹和控制。两个致命问题：(1) 花钱算力去拟合解析公式就能算的东西；(2) 黑箱决策无法通过 Safety-Critical 安全验证。

03

核心瓶颈：割裂

模型驱动强调物理却无学习能力，数据驱动强调学习却忽略物理——两者本应解决不同层次的问题，但目前彼此独立、互不融合。

04

不要选边站

未来的方向不是二选一。我们需要重新定义物理模型和神经网络之间的关系——物理负责演化，神经网络负责认知，二者深度融合而非彼此替代。

可扩展

系统能不断增加新智能能力（新任务/新感知/新控制），无需改变底层机制。以能力单元而非功能模块组织——插件式扩展。

09 / 23 · 架构创新
RUNTIME
传统航天软件：按功能划分模块 → N个模块产生 N² 条通信路径。PNIE 采用完全不同的组织方式。

从 Module 到 Runtime · 状态驱动架构

传统 Module 的局限

Module 是"输入→计算→输出→结束"的一次性函数。增加一个自主交会能力，可能同时修改轨道、规划、控制三个模块接口——模块间形成 N² 条点对点通信路径，复杂度随功能爆炸增长。

Runtime = 持久运行的能力单元

它不是算完就退出的函数，它在整个任务期间永不停止——持续维护自己的状态、持续更新。传统模块是"一次性工具"，Runtime 是"永久的同事"。

Runtime 的四个关键特征

持续性：任务期间永不退出。独立状态：各自维护自己的状态空间。松耦合：不直接调用对方，通过 USS 共享信息。异步：轨道每秒更新，认知十秒一次，行为事件驱动——各自独立节拍。

10 / 23 · Physics Runtime
DIGITAL PHYSICS LAYER
从根上改变：轨道传播从"被调用的算法"变为"持续的运行时"——Digital Physics Layer

Physics Runtime · 五大职责

01 动力学传播

二体→J2/J3摄动→高精度数值积分→姿态动力学。不限制传播模型实现，但所有结果具有统一数据接口——保证未来更换算法时对外一致。

02 约束维护

推进剂·推力·通信可见性·热控·姿态机动·电源——六类约束持续维护。其他 Runtime 直接从 USS 读取，无需重复计算。

03 可达域分析

不仅要知"现在在哪"，更要知"未来能去哪"。实时计算可到达轨道集合、最小燃料轨迹、最小时间轨迹、安全空间。

04 资源演化

推进剂消耗/电池SoC/太阳输入/热平衡/执行机构寿命——所有具有明确物理规律的资源状态由 Physics Runtime 统一演化。

05 · SAFETY GUARDIAN

所有候选行为必须通过四项验证才可进入 GNC 执行：动力学一致性 → 工程约束 → 安全边界 → 资源可行性。Physics Runtime 是整个 PNIE 的安全守护角色。

边界：Physics RT 不学习、不预测语义——Neural RT 不传播轨道、不修改动力学。两者仅通过 USS 共享状态。

11 / 23 · Neural Runtime
COGNITIVE LAYER
PNIE 与传统 AI for Aerospace 最大的区别所在

Neural Runtime

关注"状态意味着什么"——而非"状态是什么"

传统 AI 让神经网络去学动力学、学轨迹、学控制——前提假设是神经网络是环境的替代者。但在航天领域，这个假设是错的。轨道动力学已有成熟解析模型，不需要重新学习。真正需要的，是理解物理状态背后的语义。

01 环境认知

识别空间碎片密集区域、分析目标运行状态、判断通信链路稳定性——在物理状态之上形成结构化空间态势。

02 目标理解

两颗卫星完全相同相对状态：一颗交会，另一颗避碰。从动力学角度完全一样——从任务语义角度完全不同。

03 风险推理

碰撞风险/资源风险/任务冲突/通信中断——高层语义风险，无法通过动力学方程计算，需要认知推理。

04 持续学习

不采用"地面训练+星上部署"固定模式。任务反馈·环境变化·多智能体协同——三重更新来源。仅更新认知模型，不改变动力学。

LONG-TERM KNOWLEDGE
Neural RT 的 Experience Context = 工作记忆 · Memory RT = 长期记忆

Memory Runtime

让系统"越飞越聪明"——跨越多个任务周期的长期知识积累

Neural Runtime 解决当前任务内的认知推理。但"过去在这个轨道区域发生过什么？""历史数据显示这个目标的典型行为是什么？"——这些需要跨越多个任务周期的长期知识。

任务历史

已完成任务的完整轨迹、关键决策点、执行结果。结构化时序数据，任务结束时写入——形成可追溯的飞行履历。

目标档案

已知目标的轨道特征、行为模式、交互记录。如："这个轨道高度的 GEO 卫星，90% 保持位置保持行为，历史异常概率低于 5%。"

策略知识

不同场景下各种策略的有效性评价——知识图谱存储。哪种接近策略在什么条件下最可靠？策略执行后增量更新评分。

经验检索

向量嵌入+相似度搜索——给定当前状态，检索历史上最相似场景。检索结果以结构化上下文注入 USS，供 Neural RT 和 Behavior RT 使用。

13 / 23 · 状态中枢
USS & USR
在讲 Behavior Runtime 之前，必须先回答一个问题：四类 Runtime 各自维护的状态如何统一？

USS 统一状态空间 · 动态耦合 O(N²)→O(N)

Physics RT 有物理状态，Neural RT 有认知状态，Behavior RT 有任务状态，Memory RT 有长期知识——如果这些状态之间无法统一表达，整个系统就无法形成一致的推理。USS 就是解决这个问题的：让四类不同层次的状态在同一框架内表达、相互耦合、共同参与推理。

动态耦合——Physical State → Resource State（轨道机动→推进剂减少）→ Cognitive State → Mission State（识别风险→调整优先级）→ Physical State（新轨迹）。不是四个独立数据库，而是一个持续耦合演化的有机整体。

USR 五项原则：Single Source of Truth · 状态共享非模块通信 · 异步运行 · 状态可追溯 · 统一 Read/Write API。所有 Runtime 只与 USR 耦合——系统耦合度从 O(N²) 降至 O(N)。

DECISION ENGINE
USS 统一了所有状态——Behavior Runtime 正是在这个基础上完成决策的

Behavior Runtime

输出行为（Behavior），而非控制指令。行为是一种高于控制、低于任务的智能表达——任务="对目标卫星完成补给"→行为="接近→对准→对接→补给→撤离"→轨迹="3次机动到达对接点"→控制="推力器1,50%,120秒"。Behavior Runtime 是连接高层任务目标和底层控制系统的唯一桥梁。

15 / 23 · 行为组织
BEHAVIOR GRAPH
行为从"轨道保持"→"轨道转移"→"自主接近"→"对接"→"撤离"——谁来管理这种迁移？

Behavior Graph · 节点 + 五维迁移

Behavior Node · 行为节点

每个节点包含六个字段。以"自主接近"为例：Name=Autonomous Approach · Goal=缩短距离至200m以内 · Activation=目标已识别 AND 窗口可用 · Completion=距离<200m AND 速度匹配 · Capability=Navigation · Safety=距离≥50m, 速度≤1m/s。

Behavior Transition · 五维联合判断

不是"距离<100m 就进入下一阶段"的单阈值。需要全部五个维度满足：①物理——满足下一阶段动力学条件？②认知——环境确认安全？③任务——进入新阶段？④资源——支持继续？⑤经验——历史上该迁移有无失败记录？全部满足才触发。

持续运行 · 随时回退

获取USS→评估当前节点→判断五个迁移条件→切换→调用Capability→等待下周期。如果途中 Neural RT 检测到目标异常机动，系统从"自主接近"立即切回"安全撤离"——不是靠阈值，是靠对全局状态的持续评估。

Trajectory Planning · "怎么做"

轨迹规划

在行为已确定后，将 Behavior Intent 转化为具体满足动力学约束的轨迹和控制指令。这一层属于 Capability Library 的范畴。

输出 ΔV序列 · 推进剂预算 · 控制指令

实现 Navigation / Docking / Refueling Capability

双层分离的意义：Behavior Planner 不关心接近是用 EKF（扩展卡尔曼滤波）还是粒子滤波实现的——它只输出 Intent。Trajectory Planner 不关心任务为什么要接近——它只负责把 Intent 转成轨迹。行为层与控制层彻底解耦。这种分层与人类驾驶的决策逻辑完全一致——先决定变道，再打方向盘。

六个字段：Type（行为类型）+ Goal（目标/距离）+ Constraint（安全距离/速度上限）+ TimeWindow（最早/最晚）+ RiskPreference（conservative/balanced/aggressive）+ Budget（ΔV/推进剂配额）。描述目标而非路径——彻底解耦行为层与控制层。

✓ Transformer——第一代 Generator。上下文建模强、成熟高效。局限：不适合多模态不确定性。
≈ Diffusion——一对多生成。局限：多步去噪(10-100步)，实时性不足。离线候选+在线微调。
✗ RL——学长期收益但奖励函数设计困难（安全/燃料/时间难统一量化），策略黑箱。辅助优化，不作主决策器。
✗ World Model——动力学全由NN学，忽略已知物理规律。需要 Physics-guided 改造。

Physics-guided World Model

状态转移 = 物理模型(确定性) + 神经网络(残差 + 语义)
传统 World Model 的动力学完全由神经网络学习——对航天任务而言大量规律已有解析模型，重新学习既无必要也降低可信度。Physics-guided World Model 将状态转移拆分为两部分：物理模型负责确定性演化（轨道动力学、姿态动力学、环境模型），神经网络仅补偿模型残差和未知语义（未知扰动、执行器误差、未建模动力学）。$c_t$ 为 Neural Runtime 提供的环境认知上下文。
可解释

低数据需求

安全保证
Physics RT 硬约束——不可产生非法预测
物理外推
动力学提供泛化边界——不依赖训练数据覆盖
不是"让神经网络学物理"，而是"利用物理，增强学习"——第一代：Transformer + Physics RT 分层实现
